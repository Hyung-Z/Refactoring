# 0. 스파게티 코드의 개선 필요성

기존에는 페이지 하나 하나 만들어 붙히는 재미에 제작하였지만, 이를 관리 보수 하기 위해 다시 들여다보니 변수 이름도 제각각이고, 코드의 중복도 많이 존재하는 등 불필요한 로드가 많다고 여겨 개선하기로 함.

1. 기본 레이아웃은 js 파일로 분리
2. 공통 로직은 따로 분리


# 1. UI 개선 

- 개선 사유
기존의 단순 그리드/플렉스 박스 방식의 진열은 그 수가 많아짐에 따라 번잡하고 미적으로 좋지 않은 결과를 낳았기에 개선이 필요하다 생각됨


- 개선점 
0. 공통 : 헤더와 푸터

1. 메인 화면 : 무한 캐러셀 도입

무한 캐러셀을 통해 일관적인 사이즈를 유지할 수 있음
또한 반응형 환경에서도 일관적인 경험을 할 수 있도록 개선 (260119)

2. 게임화면 : 햄버거 UI 도입

이전에도 있었으나, 각 페이지마다 설정되고, 공통되지 않아 일관성이 떨어지는 UI를 통일하고 공통 로직으로 포함. (260121)
